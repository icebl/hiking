# 技术选型对比（iOS-only）

> 所属：徒步APP · 技术选型阶段
> 版本：v0.1 · 2026-06-24
> 前提：目标平台 **仅 iOS**；离线优先；自建 OSM 矢量底图 + 等高线 + 卫星栅格；后台记录是生死线；三期接云。

---

## 0. 要定的东西

| # | 决策项 | 候选 |
|---|--------|------|
| A | 开发框架 | 原生 Swift / Flutter / React Native |
| B | 地图引擎 | MapLibre Native / Mapbox / Apple MapKit |
| C | 离线瓦片格式 | PMTiles / MBTiles |
| D | 等高线 & 底图生成工具链 | planetiler + GDAL + tippecanoe |
| E | 本地存储（云同步预留） | SQLite/GRDB / SwiftData / Realm |
| F | 定位 & 传感器 | Core Location + CoreMotion |
| G | GPX 解析 | CoreGPX |

---

## A. 开发框架（最关键）

| 维度 | 原生 Swift（SwiftUI/UIKit） | Flutter | React Native |
|------|------|------|------|
| 后台定位可靠性 | **最强**，直接用 Core Location，行为可控 | 依赖插件，长时后台稳定性需验证 | 插件生态弱，后台定位坑多 |
| 气压计/传感器 | **直接** CoreMotion | 需插件/自写 channel | 需插件/自写 |
| 地图渲染性能 | **最佳**（MapLibre 原生 SDK） | 好（MapLibre 插件，偶有 platform view 开销） | 一般 |
| 离线/文件/系统分享 | 原生 API 最顺 | 可用，偶需写原生 | 可用，偶需写原生 |
| 开发效率（单 iOS） | 高（不用考虑跨端） | 高 | 中 |
| 学习成本 | 若不熟 Swift 有曲线 | 若熟 Dart/前端较低 | 若熟 JS/RN 最低 |
| 未来扩 Android | 要重写 | **几乎免费** | 较省 |

**结论 / 推荐：**
- **iOS-only + 后台定位是生死线 + 要用气压计 → 原生 Swift 最稳妥**，对定位、传感器、后台、地图性能掌控最直接。
- **但**：若你的移动端背景是 Flutter/RN/前端、且未来大概率要扩 Android，**Flutter + MapLibre 插件**是合理取舍（需提前做"后台连续定位"打样验证，这是 Flutter 方案唯一的真风险点）。
- React Native 在"长时后台定位"这类场景生态最弱，**本项目不推荐**。

> 👉 这一项取决于你的技术背景，见文末"需你确认"。

---

## B. 地图引擎 → 推荐 MapLibre Native (iOS)

| 引擎 | 矢量瓦片 | 自建/离线瓦片 | 等高线可独立开关 | 卫星栅格叠加 | 费用 |
|------|:---:|:---:|:---:|:---:|---|
| **MapLibre Native** | ✓ | ✓（PMTiles/MBTiles） | ✓（style layer 控制） | ✓ | **免费开源** |
| Mapbox | ✓ | ✓ | ✓ | ✓ | 商业，需 token/计费 |
| Apple MapKit | ✗（不支持自定义矢量） | 仅栅格叠加(MKTileOverlay) | ✗ | △ | 免费但能力受限 |

**结论**：**MapLibre Native**。开源免费、原生支持 MVT 矢量瓦片 + 栅格叠加 + 通过 style 自由开关图层（等高线开关、卫星切换全靠它），完美匹配"混合瓦片 + 等高线可开关 + 多图层切换"的需求。Flutter 方案则用 `maplibre` Flutter 插件，同一底层。

---

## C. 离线瓦片格式 → 推荐 PMTiles（矢量）+ 栅格瓦片缓存（卫星）

| | PMTiles | MBTiles |
|---|---|---|
| 结构 | **单文件**，HTTP Range/本地直读，无需 SQLite | SQLite 数据库文件 |
| 按区域下载 | **天然适合**（一个区域一个文件，易管理/删除） | 可行，需读写 SQLite |
| MapLibre 支持 | ✓（pmtiles protocol） | ✓ |
| 成熟度 | 新但已广泛使用 | 老牌、工具链全 |

**结论**：
- **底图 + 等高线（矢量）→ PMTiles**：单文件、按区域下载/删除最干净。
- **卫星影像（栅格 T_Google）→ 本地瓦片缓存**（XYZ/SQLite 缓存目录），仅原型自用，发布前换授权影像。

---

## D. 数据生成工具链（离线包怎么造）

- **底图矢量瓦片**：用 **planetiler**（吃 OSM `.osm.pbf` → 直接产出矢量瓦片/PMTiles，快）。
- **等高线**：开放 DEM（**Copernicus GLO-30**）→ GDAL `gdal_contour` 生成等高线矢量 → **tippecanoe** 切成 MVT → 打成 PMTiles。
- **打包/分区**：按地区切成多个 PMTiles，供 APP 在线下载、离线使用。
- 这套全是**离线预处理**（在你的电脑/服务器跑），APP 只消费产物。

---

## E. 本地存储（为云同步预留） → 推荐 SQLite + GRDB

| 方案 | 控制力 | 同步预留友好度 | 备注 |
|------|:---:|:---:|---|
| **SQLite + GRDB** | **高** | **高**（自定义同步字段/迁移容易） | Swift 上体验极佳，推荐 |
| SwiftData | 中 | 中 | iOS 17+，新、较省事但控制力弱 |
| Realm | 中 | 高（自带 Atlas Device Sync） | 三期同步可省力，但绑生态、增依赖 |

**结论**：**GRDB/SQLite**。轨迹点动辄上万，SQLite 存储/查询高效；同步字段（稳定 UUID、created/updated 时间戳、软删除、is_synced）自己加最灵活，契合"MVP 预留数据层"的决策。
> 备选：若希望三期同步少写后端，可评估 Realm + Atlas Device Sync，但会绑定厂商。

---

## F. 定位 & 传感器（原生能力，框架无论选谁都要用到）

- **Core Location**：GPS 定位、后台连续定位（`allowsBackgroundLocationUpdates`）、`Always` 权限、精度/距离过滤（对应采样 5s + 位移 5m）。
- **CoreMotion `CMAltimeter`**：气压计相对高度（设备支持时用作海拔优先源，对应记录模块决策）。
- 偏航/距离计算：纯几何运算，自实现即可（投影到折线、球面距离）。

---

## G. GPX 解析 → 推荐 CoreGPX

- **CoreGPX**：Swift 的 GPX 读写库，支持 `trk/trkseg/trkpt/wpt/ele/time`，导入导出都能用，省去手写解析。
- Flutter 方案：用 `gpx` Dart 包，或在原生侧解析。
- 容错（缺 ele/time、超大文件、多 trk）仍需在业务层补，见导入模块。

---

## 推荐技术栈（一套，方案 A：原生）

| 层 | 选择 |
|----|------|
| 框架 | **Swift + SwiftUI**（必要处用 UIKit） |
| 地图 | **MapLibre Native iOS** |
| 离线格式 | **PMTiles**（矢量底图/等高线）+ 栅格瓦片缓存（卫星） |
| 数据生成 | planetiler（底图）+ GDAL gdal_contour + tippecanoe（等高线）→ PMTiles |
| 本地库 | **SQLite + GRDB**（预留同步字段） |
| 定位/传感器 | Core Location + CoreMotion |
| GPX | CoreGPX |
| 三期后端 | `End` 目录，账号 + 轨迹云同步（技术待三期定） |

**方案 B（跨端，若你更熟 Flutter / 未来要 Android）**：Flutter + `maplibre` 插件 + 同款 PMTiles/工具链 + drift(SQLite) + gpx 包 + 原生 channel 兜底后台定位。**前置风险**：必须先验证 Flutter 长时后台连续定位的稳定性。

---

## 主要风险点

1. **后台连续定位耗电与稳定性**（任何框架都要实测 4–8 小时）。
2. **卫星图层版权**（T_Google 仅原型，发布前换）。
3. **等高线数据量**：全国预生成体积大，建议按区域下载。
4. **Flutter 方案**：后台定位需打样验证后再定。

---

## 最终选型（已定稿）

- ✅ **框架：原生 Swift + SwiftUI**（必要处用 UIKit）。
- ✅ **地图引擎：MapLibre Native iOS**。
- ✅ **离线格式：PMTiles**（矢量底图/等高线）+ 栅格瓦片缓存（卫星）。
- ✅ **数据生成：planetiler（底图）+ GDAL gdal_contour + tippecanoe（等高线）→ PMTiles**。
- ✅ **本地库：SQLite + GRDB**（预留同步字段：UUID / created / updated / 软删除 / is_synced）。
- ✅ **定位/传感器：Core Location + CoreMotion（CMAltimeter 气压计）**。
- ✅ **GPX：CoreGPX**。
- 三期后端（`End`）：账号 + 轨迹云同步，技术待三期定。
