# 模块细化：沿轨迹导航（轨迹跟随）

> 所属：徒步APP PRD · 第 3.2 节展开
> 版本：v0.1 · 2026-06-24
> 定位：**轨迹跟随 + 偏航提醒**，不是驾车那种转向算路导航。

---

## 1. 这个模块要解决的核心问题

用户手上有一条线（自己存的或导入的 GPX），他要**照着这条线走**。APP 要持续告诉他三件事：

1. 我**在不在线上**？偏了要提醒。
2. 我**走了多少、还剩多少**（距离 / 爬升）？
3. 前方**有没有要注意的点**（岔路、水源、营地）？

---

## 2. 进入导航的流程

```
轨迹库/轨迹详情 → 点「沿此轨迹徒步」
   → 选择方向：正向 / 反向（轨迹有方向，反着走要反向）
   → 选择是否同时记录我的实际轨迹（默认：是）
   → 进入导航地图页
```

**关键点：**
- **方向选择是必须的**：很多轨迹是单程线，反向走时所有"剩余距离/前方航点"都要按反向重算。
- **同时记录**：导航时一般要把"我实际怎么走的"也记下来（计划线 vs 实走线，事后可对比）。默认开启，复用轨迹记录模块。

---

## 3. 导航地图页要显示什么

| 元素 | 说明 |
|------|------|
| 计划线 | 醒目颜色（如粗蓝线），与底图区分 |
| 已走/未走 | 计划线按当前进度分段着色：已走变灰，未走高亮（P1） |
| 我的位置 + 朝向 | 箭头表示设备朝向；定位差时变灰/加圈 |
| 起点/终点标记 | |
| 沿线航点 | 岔路、水源、营地等图标 |
| 实走轨迹 | 若开启记录，红色实走线叠加在计划线上（P1） |
| 信息条 | 剩余距离、剩余爬升、（P1）ETA、（P1）到下一航点距离 |
| 状态提示 | 在线 / 偏航 / 定位差 / 已到达 / 未上线 五种状态 |
| 重新居中按钮 | 自由拖动地图后，一键回到当前位置 |

相机模式：**自动跟随**（地图随人移动、可选随朝向旋转）/ **自由浏览**（手动拖动后停止跟随，显示居中按钮）。

## 4. 偏航检测（本模块技术核心）

### 4.1 基本算法
- 把当前 GPS 点投影到计划线（折线）上，求**到最近线段的垂直距离** `d`。
- `d > 偏航阈值` → 进入"偏航"状态。

### 4.2 防抖（必须，否则会反复误报）
- **进入偏航**：`d` 持续超过阈值 **且** 持续 N 秒（如连续 10s）才判定偏航，避免 GPS 抖一下就报警。
- **解除偏航**：回到 `d < 解除阈值`（解除阈值 < 偏航阈值）才算回到线上（滞回，防止在阈值边缘反复横跳）。
- **默认偏航阈值 25m**（解除阈值取约 15m），**用户可在设置中自定义修改**。高山密林可调大。

### 4.3 提醒方式
- 进入偏航瞬间：**默认 = 系统通知 + 震动**（不含语音播报）。
- **可选语音播报**：用户可在设置中开启"持续语音播报"，并选择播报间隔（**5 分钟 / 10 分钟**等），偏航持续期间按该间隔重复播报。默认关闭（费电）。
- **关键：后台/锁屏也要能提醒**（徒步时手机多在兜里），靠系统通知 + 声音/震动，依赖后台定位。
- 重复提醒节流：偏航持续时不要一直响，按状态变化提醒；开启语音播报后按所选间隔重复。
- 静音/勿扰场景给开关（声音、震动可分别关）。

### 4.4 自交轨迹 / 回头路的处理
- 折线有回折（上山下山同一条路、之字形上升、环线）时，"最近点"会有歧义。
- 解决：维护**当前进度索引**，最近点搜索只在"上次匹配位置前后一个窗口"内进行（单调推进），避免一下跳到对面的线段上。

## 5. 进度与剩余信息

- 把当前位置投影到线上 → 得到**已行进里程**（沿线距离，非直线距离）。
- 预计算：计划线每个点的**累计距离**、**累计爬升**（导入时算好，缓存）。
- 剩余距离 = 总里程 − 已行进里程；剩余爬升同理。
- ETA（P1）：基于近期平均速度估算，山路误差大，标注"仅供参考"。
- 到下一航点距离 / 方向（P1）。

---

## 6. 航点提示

- 接近沿线航点（如 < 100m）→ 通知"前方 80m：水源点"。
- 岔路口航点可标注"计划线走左 / 右"，帮助在路口不走错（P1）。

---

## 7. 边界情况清单（重点，逐项要有行为定义）

| 情况 | 期望行为 |
|------|----------|
| GPS 信号差/精度低 | **暂停偏航判定**（精度差于阈值时不报警，避免误报），状态显示"定位较差"，恢复后再判定 |
| 出发点离起点很远 | 不立即报偏航；提示"距起点 X km，导航将在你接近线路后开始判定"；上线后才启用偏航逻辑 |
| 反向导航 | 剩余距离/爬升、前方航点全部按反向重算 |
| 环线（起=终） | 进度按推进方向单调计算，回到起点附近判定"已到达" |
| 原路往返（out-and-back） | 去程用前进窗口，到折返点后切换搜索方向 |
| 多段轨迹中间有断点 | 跨断点时不报偏航，提示"进入下一段" |
| 到达终点 | 通知"已到达终点"，**仅提示，不自动结束**；导航与记录继续，由用户手动停止 |
| 长时间静止（休息） | 不触发任何异常；记录模块可自动暂停（与记录模块联动，P1） |
| APP 被系统杀死 | 重启后能恢复"正在导航 + 正在记录"状态（复用记录模块的崩溃恢复） |

## 8. 状态机（建议）

```
未上线 ──接近并贴近计划线──▶ 在线
在线 ──偏离>阈值且持续Ns──▶ 偏航（震动+音+通知）
偏航 ──回到<解除阈值──▶ 在线
任意状态 ──精度差于阈值──▶ 定位较差（暂停偏航判定）
定位较差 ──精度恢复──▶ 回到之前状态
在线/偏航 ──到达终点判定──▶ 已到达
```

---

## 9. 数据与计算需求

- 计划线：点序列（经纬度 + 海拔），来自轨迹库。
- 预计算并缓存：逐点累计距离、累计爬升、（可选）每段方位角。
- 最近点查询：用"上次索引 ±窗口"的局部搜索即可，无需重型空间索引（轨迹点量级不大）。
- 高度数据：优先用计划线自带海拔；缺失时可用离线 DEM 采样补（与等高线模块共用 DEM）。
