# 模块细化：导入 / 导出 / 分享

> 所属：徒步APP PRD · 第 3.5 / 3.6 节展开
> 版本：v0.1 · 2026-06-24
> 定位：让轨迹能进得来、出得去。MVP 阶段分享 = 文件分享，不依赖后端。

---

## 1. 这个模块要解决的核心问题

- **导入**：把别人/其他 APP 的轨迹文件读进来，存入本地轨迹库，可查看、可作导航线。
- **导出**：把本地轨迹存成标准文件。
- **分享**：通过系统分享面板把文件发给微信/邮件/其他 APP（不做云端链接，那是三期）。

---

## 2. 支持格式

| 格式 | 导入 | 导出 | 优先级 | 说明 |
|------|:---:|:---:|:---:|------|
| GPX (1.1) | ✓ | ✓ | **P0** | 户外通用格式，首要支持 |
| KML | ✓ | ✓ | P1 | Google 系，含样式 |
| KMZ | ✓ | ✓ | P1 | KML 的压缩包（可能含图片） |

GPX 元素对齐：`trk/trkseg/trkpt`（轨迹）、`wpt`（航点）、`rte`（路线，可选）、`ele`（海拔）、`time`（时间）。

---

## 3. 导入流程

```
来源 → 选择/接收文件 → 解析校验 → 预览 → 确认导入 → 存入轨迹库
```

### 3.1 导入入口（来源）
- **系统“用本应用打开”**：从文件管理器、微信、QQ、邮件等点 .gpx 文件，选择用本 APP 打开（P0）。
- **系统分享面板接收**：别的 APP 分享 .gpx 给本 APP（P0）。
- **APP 内文件选择器**：在 APP 里浏览本地文件导入（P0）。

### 3.2 解析与校验（容错是重点）
- 解析 GPX，提取轨迹点、航点、海拔、时间。
- **容错处理**（真实文件经常不规范）：
  - 缺 `ele`（无海拔）：导入后可用 DEM 采样补海拔。
  - 缺 `time`（无时间戳）：仍可作为路线使用，只是无时间统计。
  - 编码/换行/命名空间不规范：尽量兼容解析。
  - 超大文件（几万点）：分段解析，避免卡 UI。
  - 多 `trk`/多段：保留分段结构。
  - 损坏/非法文件：明确报错，不崩溃。

### 3.3 预览与确认
- 解析后先**预览**：地图显示轨迹 + 距离/爬升等统计。
- 用户确认后存入轨迹库，可编辑名称、加备注。
- **重复导入检测**（P1）：同一文件/高度相似轨迹提示“可能已存在”，避免库里重复。

---

## 4. 导出流程

```
轨迹详情 → 导出 → 选择格式(GPX/KML) → 生成文件 → 系统分享面板
```

- 选择导出格式（MVP：GPX）。
- 选择导出内容：仅轨迹 / 轨迹 + 航点（P1）。
- 生成标准文件，命名建议含轨迹名 + 日期。
- 可一次导出多条（批量，P1）。

---

## 5. 分享流程（MVP = 文件分享）

- 导出生成的文件 → 调起**系统分享面板** → 发给微信/邮件/网盘/其他 APP。
- 接收方用本 APP（或任意支持 GPX 的 APP）打开即可导入。
- **不做**云端链接分享、二维码分享（三期，依赖后端）。

---

## 6. 边界情况清单

| 情况 | 期望行为 |
|------|----------|
| GPX 无海拔 | 正常导入；提示可用 DEM 补海拔 |
| GPX 无时间 | 正常导入；时间相关统计置空 |
| 文件损坏/格式错误 | 明确报错说明原因，不崩溃 |
| 超大轨迹文件 | 分段解析 + 进度提示，不卡死 |
| 含多条轨迹的文件 | **MVP：不提供选择界面，全部自动导入为多条独立轨迹**（已定） |
| 仅含航点无轨迹 | 作为航点集合导入（P1） |
| 坐标系异常（疑似 GCJ-02 偏移） | GPX 标准应为 WGS-84；若检测到明显偏移给出提示（难判定，尽力而为） |
| 导出无存储权限 | 申请权限或提示 |

---

## 7. 数据模型映射

- 导入：GPX/KML → 内部 Track / Waypoint 模型（见 PRD 第 6 节），字段一一对应。
- 导出：内部模型 → 标准 GPX 1.1，保证被其他主流户外 APP（两步路/六只脚/佳明等）正确读取。
- **互通性测试**：导出的文件要能被对标 APP 正常打开，导入要能吃下它们导出的文件。

---

## 8. 本模块决策记录

**已定：**
- ✅ **MVP 只做 GPX**；KML/KMZ 放 P1。
- ✅ **导入去重放 P1**（MVP 不做重复检测）。
- ✅ **多轨迹文件**：MVP 不提供选择界面，全部自动导入为多条独立轨迹。
- ✅ **航点（`wpt`）随轨迹一起导入导出**（MVP 即包含，不丢弃别人轨迹中的营地/水源/岔路等航点）。
